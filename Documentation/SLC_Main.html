LANDFORMS 01 terrain variables

This tool generates the four key variables required to perform the surface elements classification: slope, ruggedness (vrm), finescale and broadscale bathymetric position index (BPI).

Slope calculates the gradient change of the surface in degrees. Slope is calculated using the QGIS GDAL 'Slope' tool (GDAL/OGR Contributors, 2025).

Ruggedness (vrm) calculates the variation of a surface as a representation of roughness, with output values ranging from 0 (no variation) to 1 (complete variation). Ruggedness is calculated using the SAGA 'Vector Ruggedness Measure (VRM)' tool (Conrad et al., 2015) and uses a 'square' search mode with radius 1.

Bathymetric Position Index (BPI) calculates relative elevation across a seascape, where large positive values represent higher elevation areas of the dataset, large negative values represent area of lower elevation, and values around zero represent flatter areas. BPI is calculated using the methodology from the Geomorphometry and Gradient Metrics v2.0 toolbox 'Slope Position' tool (Evans et al. 2014) and the WhiteBox Tools 'Median Filter' tool (Lindsay, 2016). BPI grids are calculated at user-defined broad and fine scales. Broadscale BPI uses a higher value for the window size, while finescale BPI uses a lower value for the window size.

Default values for the BPI window sizes are based on a 5 m DEM (with no smoothing performed) and are intended as a guide.

This tool is intended for users to explore the terrain variables and determine suitable threshold settings.

Input parameters

DEM

Input raster dataset (DEM) as a GeoTIFF.

Region Label

Name for output directory (TERRAIN_[Region Label]) that results will be transfered to i.e. test, that is created if necessary. All transfered results are also renamed with this label as a prefix i.e. test_slope.tif.

BroadBPI Window Size

This variable specifies the window size (distance in cells) that is used to calculate broadscale BPI.

For example, a threshold of 150 cells will use a 150 cell-radius (750 m for a 5 m cell DEM) from an individual cell to calculate relative elevation. This 150-cell window will be run across the entire dataset to calculate the relative high, flat and low areas of the dataset at a broad scale.

Default values for the BPI window sizes are based on a 5 m (with no smoothing performed) and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

FineBPI Window Size

This variable specifies the window size (distance in cells) that is used to calculate finescale BPI.

For example, a threshold of 27 cells will use a 27 cell-radius (135 m for a 5 m cell DEM) from an individual cell to calculate relative elevation. This 27-cell window will be run across the entire dataset to calculate the relative high, flat and low areas of the dataset at a broad scale.

Default values for the BPI window sizes are based on a 5 m (with no smoothing performed) and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

Outputs

Outputs GeoTIFF rasters to ../Working/ for slope, vrm, broadbpi and finebpi which are subsequently renamed and transfered to the TERRAIN_[Region Label] directory.

LANDFORMS 02 surface elements

The surface elements classification breaks up the surface into key components based on ruggedness, slope, finescale and broadscale bathymetric position index (BPI). The user is required to enter classification thresholds for each variable, which forms the resultant surface elements. The approach has been developed to target shelf reef features, which are captured effectively by the ruggedness (VRM) variable.

The default settings have been determined based on a 5 m cell size raster, with no smoothing performed. Settings should be modified by the user based on the resolution and features of interest within each dataset (see Table 1). The settings applied are recorded as a text file prefixed with 'Current Session'.

Tip: Users may explore different threshold settings by classifying the symbology within the layer's properties.

The surface elements classification characterises the surface into 11 detailed classes of surface texture, which may combine one or two reclassified terrain variables. For example, 'Broadscale high Rugose' surface element class combines the user-defined 'high' broadscale BPI class and 'rugose' ruggedness class. The 'surface element' labels are defined in the output 'SURF_EL' field.

The 11 'surface element' classes are then aggregated into 7 classes to create a 'summarised surface elements' layer, which offers a simplified layer for users to interpret features. For example, the 'Broadscale high Rugose' would be aggregated with other rugose 'high' classes to form 'Rugose outcrops'. The 'summarised surface element' labels are defined in the output 'SUM_SE' field.

The results of the tool are renamed and transferred to the SURFEL_[Region Label] directory.

Slope Threshold

Slope calculates the gradient change of the surface in degrees.

Slope is calculated using the QGIS GDAL 'Slope' tool (GDAL/ORG Contributors, 2025).

User-defined thresholds will reclassify the data into 'slope' (higher slope) classes. The extent of higher slope areas will be carried through to the surface elements classification as 'Slope' classes. These classes will ultimately be defined as 'Scarps' in the landform classification.

For example, a slope threshold of 10 degrees will reclassify: 1) all values ≥ 10 as 'slope' surface element classes; and 2) all values < 10 degrees as lower slope areas, not defined in the surface elements and landform classifications. All values < 10 degrees are instead defined by other surface element classes (ruggedness, finescale BPI, broadscale BPI).

Default values are based on a 5 m DEM (with no smoothing performed) and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

Ruggedness Threshold

Ruggedness (vrm) calculates the variation of a surface as a representation of roughness, with output values ranging from 0 (no variation) to 1 (complete variation).

Ruggedness is calculated using the SAGA 'Vector Ruggedness Measure (VRM)' tool (Conrad et al., 2015) and uses a 'square' search mode with radius 1.

A user-defined threshold will reclassify the data into 'rugose' (higher ruggedness) and 'smooth' (lower ruggedness) classes. The extent of rugose outcrops defined will be carried through to the surface elements classification as 'Rugose' classes and, for example, may become a 'Reefs/Banks' class in the landform classification. Areas below the selected ruggedness value for rugose features will be carried through as 'Smooth' classes in the surface elements classification and, for example, may become a 'Plain' class in the landform classification.

For example, a ruggedness threshold of 0.0002 will reclassify: 1) all values ≥ 0.0002 as 'rugose' surface element classes; and 2) values < 0.0002 as 'smooth' surface element classes.

Default values are based on a 5 m DEM smoothed (with no smoothing performed) and are intended as a guide. Ruggedness threshold values should be adjusted according to input DEM resolution and smoothing iterations (if applied). See Table 1 for examples.

Individual datasets should be interrogated to determine the most suitable threshold settings.

Tip: When determining the ruggedness threshold, it is important to note that subsequent steps in the classification methodology will identify polygons within a rugose outcrop. If the rugose outcrop edge is adequately captured by the ruggedness threshold, 'gaps' within the outcrop will be identified in later steps as polygons within the outcrop surface. To select a suitable ruggedness threshold, the extent of rugose outcrops captured needs to be balanced with the amount of noise captured, and in turn, manual editing required. Noise correction steps applied in later steps of the landform classification reduce noise within the dataset. See the 'preliminary landforms' section for further information on the noise correction procedure.

BroadBPI Thresholds

Bathymetric Position Index (BPI) calculates relative elevation across a seascape, where large positive values represent higher elevation areas of the dataset, large negative values represent area of lower elevation, and values around zero represent flatter areas.

BPI is calculated using the methodology from the Geomorphometry and Gradient Metrics v2.0 toolbox 'Slope Position' tool (Evans et al., 2014) and the WhiteBox Tools 'Median Filter' tool (Lindsay, 2016).

Two user-defined thresholds will reclassify the data into 'broadscale low' (negative BPI values), 'flat' (BPI values around zero), and 'broadscale high' (positive BPI values) classes.

The upper limit of broadscale BPI, will create 'Broadscale High' classes within the surface elements classification, and the lower BPI limit of broadscale BPI, will create 'Broadscale Low' classes. The values in between will be defined as 'Flat' areas. 'Broadscale High' surface elements, for example, may become 'Peak' classes in the landform classification, and 'Broadscale Low' classes, for example, may become 'Depressions and channels' in the landform classification.

For example, a threshold of -100; 100 will create a reclassified polygon separated into three classes: 1) the lower extent of data values (dataset-dependent) to -100. These larger, negative BPI values as classed as 'broadscale low'; 2) -100 to 100. These data values around zero are classed as 'flat'; 3) 100 to the upper extent of data values (dataset-dependent). These larger, positive BPI values are classed as 'broadscale high'.

Default values are based on a broadscale BPI window size of 150 cells with 5 m DEM (with no smoothing performed) and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

LANDFORMS 03 prelim landforms

Seabed landforms are defined from the classified surface elements, where multiple surface element classes may be ultimately grouped to form a landform. Landforms represent key features of the seascape, including 'reefs/banks', 'peaks', 'plains', 'scarps', 'depressions and channels'. These classes do not represent the only landforms that may occur within a seascape and are intended to capture the prominent features observed within a continental or island shelf setting.

The user is required to input the 'surface elements' created in LANDFORMS 02 and the associated DEM. A user-defined ruggedness threshold is required which will be used to identify potential 'noise' polygons.

This tool first defines preliminary landforms. The user is required to review and edit these classes before proceeding to finalise the classification.

All classes should be reviewed, however specific classes are flagged with a 'REVIEW' label where they may fall into one of two categories. For example, 'Depressions and channels Rugose REVIEW' may be 'Depressions and channels Rugose' or 'Reefs/Banks'.

Tip: It is recommended the user display the slope grid transparently (e.g. 50%) over the preliminary landforms classification. The input DEM should also be loaded for interrogation.

The 'prelim_landforms' feature class is the preliminary landforms classification output which requires manual review and potential editing by the user. Preliminary landform labels are attributed within the field "PrelimLand". These include: Peaks, Peaks REVIEW, Reefs/Banks, Reefs/Banks REVIEW, Scarps, Depressions and channels Smooth, Depressions and Channels Rugose, Depressions and Channels Rugose REVIEW, and Plains. 'Rugose' and 'Smooth' qualifiers are provided for the depressions and channels. 'Rugose' depressions and channels are considered part of the outcropping reef/bank surface, whereas 'Smooth' depressions and channels are contained by, rather than a part of the outcropping reef/bank surface. Note: the 'LAND_2' field is a duplicate of 'PrelimLand' in case the user needs to refer back to the original labels when performing edits.

The landform classification approach is flexible to the user's specific terminology of features and users can add landform terms. The toolset provides classified boundaries which can be reviewed and altered by the user to create the final classification. Any new or altered terms applied during the editing of the preliminary landforms layer will be carried across as final landform labels.

Tip: If new landform labels are defined, these will not be visible in the default Symbology associated with the shapefile.

Required edits:

Several classes are labelled with a "REVIEW" suffix, including "Depressions and channels Rugose REVIEW" and "Reefs/Banks REVIEW". They occur when a class may fall into one of two classes (see Table 2). The most likely class is assigned, however visual inspection is required to confirm or relabel the polygon into the appropriate class.

During the procedure to define preliminary landforms, 'noise' polygons are identified in areas where rugose polygons do not overlap with higher ruggedness areas. These polygons require review and potential editing by the user to ensure the full extent of the observed features are captured. The original landform terms are captured within the 'LAND_1' field, and the 'LAND_2' labels can be updated by carrying across these original ('LAND_1') labels as required.

Optional edits:

The output of the landforms classification creates an aggregated category of "reefs/banks". 'Reefs' represent inferred rock or biogenic reef features, and 'banks' are inferred as soft-sediment features (Table 3). While the landform classification is not intended to map substrates, rock/biogenic reef and soft sediment outcrops can be discernible to users familiar with the seascape (as occurs where features are manually digitised), and in such cases they can be considered distinct landforms. While these features are not separated automatically through the toolbox, users can manually edit the output to separate these features, if desired. This flexibility allows users to adjust the landform output to meet their requirements.

Additional 'Rugose' channel features are captured by the 'Depressions and channels Rugose REVIEW' class. These features can be reviewed and manually edited to be included in the final landform classification if the user desires greater detail of variation across the reef/bank surface. This class can include the reef/bank edge as a rugose 'low' depression/channel. Therefore, to separate the additional rugose channels, the user must manually edit and cut the channel extent from the 'Depressions and channels Rugose REVIEW' polygon. If this editing is not performed, the 'Depressions and channels Rugose REVIEW' classes will be labelled as 'Reefs/Banks' in the subsequent step to finalise landforms. Ruggedness Threshold

A user-defined 'noise' ruggedness threshold is used to identify potential noise polygons in the data. Noise may be erroneous data sourced from the input bathymetry (such as speckled noise in marine lidar data or imagery), or superfluous rugose outcrops. To correct for these additional 'noise' areas, a 'noise ruggedness threshold' is used to identify areas of higher ruggedness, which are inferred areas of greater relief and therefore less likely to be noise in the data.

With a user-defined 'noise' ruggedness threshold, processes are implemented to overlap the ‘ruggedness noise threshold’ (higher ruggedness value, e.g. 0.001) with the rugose outcrops defined in the surface elements classification (lower ruggedness value, e.g. 0.0002). Where rugose outcrops overlap with the ruggedness noise threshold, the preliminary landform labels are retained. Where they do not overlap, the polygons are labelled as 'noise'. These 'noise' polygons are to be manually reviewed by the user in later steps to ensure polygons are correctly attributed.

A suitable 'noise' ruggedness threshold can be identified by interrogating the ruggedness layer, and selecting a value which, when overlapped with the ruggedness threshold for rugose outcrops, will retain the most appropriate extent of the rugose outcrop. An example is provided below in Figure 7.

Default values are based on a 5 m DEM (with no smoothing performed) and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

LANDFORMS 04 final landforms

This tool finalises the preliminary landforms into the final landform classification.

This tool should be performed after manual review has been conducted on the preliminary landform classification. In this tool, 'Preliminary landform' labels ('LAND_2' field) are transferred into final landform labels ('LANDFORM' field).Unless the classes have been edited to label otherwise, "Peaks REVIEW" classes will become "Peaks", "Depressions and channels Rugose REVIEW" will become "Reefs/Banks", and "Reef/Banks REVIEW" will become "Reefs/Banks" (see Table 2).

This tool has optional functionality to eliminate small polygons, with a user-defined polygon area.

The results of the tool are renamed and transferred to the directory LANDFORM_[Region Label].

LANDFORMS 05 plain landforms

Plain landforms are defined using finescale and broadscale BPI, creating classes including Plain High, Plain Low, Localised high, Localised low and Plain flat. This classification intends to capture additional detail across the plain area for users focused on the inferred sedimentary environment.

Where an existing 'Plains DEM' doesn't exist, a plains DEM is extracted from the original DEM using the plain features classified in the final landforms layer from LANDFORMS 04 final landforms.

The user-defined BPI thresholds and window sizes will reclassify the data into 5 plain landform classes: 'localised low', 'localised high', 'plain low', 'plain flat' and 'plain high'.

This tool has optional functionality to eliminate small polygons, with a user-defined polygon area.

Default values are based on a 5 m DEM (witn no smoothing performed) and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

The results of the tool are renamed and transfered to the directory PLAIN_[Region Label].

LANDFORMS depth reclassification

Depth reclassification tool that generates polygons at depth intervals specified by the user, with depth intervals calculated from zero.

For example, an interval value of 10 m for a dataset ranging -17 to -55 m will generate polygons at -10 to -20 m, -20 to -30 m, -30 to -40 m, -40 to -50 m, and -50 to -60 m intervals.

Input parameters

DEM

Input raster dataset (DEM) as a GeoTIFF

Depth Interval

Depth interval (m) to be used to generate polygons.

DEM Label

Label for output directory i.e. PREP_[DEM Label].

If left blank reclassified the depth shapefile (DEM_depth_reclass.shp) is output to ../Working/ directory.

Outputs

A reclassified depth shapefile named DEM_[DEM_label]_depth_reclass.shp is output to PREP_[DEM Label].

If DEM Label is left blank the reclassified depth shapefile (DEM_depth_reclass.shp) is output to ../Working/ directory.
