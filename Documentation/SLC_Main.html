LANDFORMS 01 terrain variables

This tool generates the four key variables required to perform the surface elements classification: slope, ruggedness (vrm), finescale and broadscale bathymetric position index (BPI).

Slope calculates the gradient change of the surface in degrees. Slope is calculated using the QGIS GDAL 'Slope' tool.

Ruggedness (vrm) calculates the variation of a surface as a representation of roughness, with output values ranging from 0 (no variation) to 1 (complete variation). Ruggedness is calculated using the SAGA 'Vector Ruggedness Measure (VRM)' tool and uses a 'square' search mode with radius 1.

Bathymetric Position Index (BPI) calculates relative elevation across a seascape, where large positive values represent higher elevation areas of the dataset, large negative values represent area of lower elevation, and values around zero represent flatter areas. BPI is calculated using the methodology from the 'Slope Position' tool in the Geomorphometry and Gradient Metrics v2.0 toolbox (Evans et al. 2014) and is calculated at user-defined broad and fine scales. Broadscale BPI uses a higher value for the window size, while finescale BPI uses a lower value for the window size.

Default values for the BPI window sizes are based on a 5 m DEM smoothed with x3 iterations and are intended as a guide.

This tool is intended for users to explore the terrain variables and determine suitable threshold settings.

Evans, J., Oakleaf, J., Cushman, S., 2014. An ArcGIS Toolbox for Surface Gradient and Geomorphometric Modeling, Version 2.0-0. Available online: https://github.com/jeffreyevans/GradientMetrics

Input parameters

DEM

Input raster dataset (DEM) as a GeoTIFF.

BroadBPI Window Size

This variable specifies the window size (distance in cells) that is used to calculate broadscale BPI.

For example, a threshold of 150 cells will use a 150 cell-radius (750 m for a 5 m cell DEM) from an individual cell to calculate relative elevation. This 150-cell window will be run across the entire dataset to calculate the relative high, flat and low areas of the dataset at a broad scale.

Default values are based on a 5 m DEM smoothed with x3 iterations and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

FineBPI Window Size

This variable specifies the window size (distance in cells) that is used to calculate finescale BPI.

For example, a threshold of 27 cells will use a 27 cell-radius (135 m for a 5 m cell DEM) from an individual cell to calculate relative elevation. This 27-cell window will be run across the entire dataset to calculate the relative high, flat and low areas of the dataset at a broad scale.

Default values are based on a 5 m DEM smoothed with x3 iterations and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

Outputs

Outputs GeoTIFF rasters to ../Working/ for slope, vrm, broadbpi and finebpi.

LANDFORMS 02 surface elements

Make surface elements shapefile from DEM

The surface elements classification breaks up the surface into key components based on ruggedness, slope, finescale and broadscale bathymetric position index (BPI). It characterises the surface into detailed classes of surface texture which may combine one or two reclassified terrain variables.

For example, 'Broadscale high Rugose' surface element class combines the user-defined 'high' broadscale BPI class and 'rugose' ruggedness class. The 'surface element' labels are defined in the output 'SURF_EL' field.

The surface elements classification outputs 11 classes which represents the full suite of component elements. These are aggregated into 7 classes to create a 'summarised surface elements' layer, which offers a simplified layer for users to interpret features. For example, the 'Broadscale high Rugose' would be aggregated with other rugose 'high' classes to form 'Rugose outcrops'. The 'summarised surface element' labels are defined in the output 'SUM_SE' field.

Input parameters

DEM

Input raster dataset (DEM) as a GeoTIFF

Region Label

Name for output directory that results will be transfered to i.e. test, that is created if necessary. All transfered results are also renamed with this label as a suffix i.e. test_surf_elem.shp.

Slope Threshold

Slope calculates the gradient change of the surface in degrees. Here, the user will select a value to define higher slope areas, which will be carried through to the surface elements classification as 'Slope' classes. These classes will ultimately be defined as 'Scarps' in the landform classification.

User-defined thresholds will reclassify the data into 'slope' (higher slope) classes.

For example, a slope threshold of 10 degrees will reclassify: 1) all values >= 10 as 'slope' surface element classes; and 2)all values < 10 degrees as lower slope areas, not defined in the surface elements and landform classifications. All values < 10 degrees are instead defined by other surface element classes (ruggedness, finescale BPI, broadscale BPI).

Default values are based on a 5 m DEM smoothed with x3 iterations and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

Slope is calculated using the QGIS GDAL ‘Slope’ tool.

Ruggedness Threshold

Ruggedness (vrm) calculates the variation of a surface as a representation of roughness, with output values ranging from 0 (no variation) to 1 (complete variation). Here, the user will select a value to define the extent of rugose outcrops, which will be carried through to the surface elements classification as 'Rugose' classes and, for example, may become a 'Reef/Bank' class in the landform classification. Areas below the selected ruggedness value for rugose features will be carried through as 'Smooth' classes in the surface elements classification and, for example, may become a 'Plain' class in the landform classification.

A user-defined threshold will reclassify the data into 'rugose' (higher ruggedness) and 'smooth' (lower ruggedness) classes.

For example, a ruggedness threshold of 0.00005 will reclassify: 1) all values >= 0.00005 as 'rugose' surface element classes; and 2) values < 0.00005 as 'smooth' surface element classes.

Default values are based on a 5 m DEM smoothed with x3 iterations and are intended as a guide. Input DEM resolution and smoothing alters the output ruggedness, and threshold values should be adjusted accordingly.

Individual datasets should be interrogated to determine the most suitable threshold settings.

Ruggedness is calculated using the SAGA 'Vector Ruggedness Measure (VRM)' tool and uses a 'square' search mode with radius 1.

BroadBPI Thresholds

Bathymetric Position Index (BPI) calculates relative elevation across a seascape, where large positive values represent higher elevation areas of the dataset, large negative values represent area of lower elevation, and values around zero represent flatter areas. Here, the user will select two values to define an upper limit of broadscale BPI, which will create 'Broadscale High' classes within the surface elements classification, and a lower BPI limit of broadscale BPI, which will create 'Broadscale Low' classes. The values in between will be defined as 'Flat' areas. 'Broadscale High' surface elements, for example, may become 'Peak' classes in the landform classification, and 'Broadscale Low' classes, for example, may become 'Depressions and channels' in the landform classification.

Two user-defined thresholds will reclassify the data into 'broadscale low' (negative BPI values), 'flat' (BPI values around zero), and 'broadscale high' (positive BPI values) classes.

For example, a threshold of -100; 100 will create a reclassified polygon separated into three classes: 1)the lower extent of data values (dataset-dependent) to -100. These larger, negative BPI values as classed as 'broadscale low'; 2)-100 to 100. These data values around zero are classed as 'flat'; 3)100 to the upper extent of data values (dataset-dependent). These larger, positive BPI values are classed as 'broadscale high'.

Default values are based on a broadscale BPI window size of 150 cells with 5 m DEM smoothed with x3 iterations and are intended as a guide.

Individual datasets should be interrogated to determine the most suitable threshold settings.

BPI is calculated using the methodology from the 'Slope Position' tool in the Geomorphometry and Gradient Metrics v2.0 toolbox (Evans et al. 2014)

Evans, J., Oakleaf, J., Cushman, S., 2014. An ArcGIS Toolbox for Surface Gradient and Geomorphometric Modeling, Version 2.0-0. Available online: https://github.com/jeffreyevans/GradientMetrics
