PREP clip DEM

Clip DEM raster to specified depths.

An upper and lower elevation value can be set by the user to clip the DEM. The values used to clip the dataset will be added as a suffix to the output dataset, e.g. dem_10 if the dataset was clipped at -10 m elevation. Input values must be positive. The Seabed Landforms Toolset has been developed for open coast, shelf settings and is not designed for land or estuarine environments. It is recommended all DEMs be clipped to 0 m elevation for the upper extent of data to represent bathymetry only, and estuarine environments be excluded unless determined by the user as appropriate (Figure 1).

Furthermore, the lower depth extent of the dataset should be considered where there is a large variation in elevation. If the dataset extends beyond the continental or island shelf to include the shelf break, shelf slope and/or abyssal plain, it is strongly recommended the lower extent of elevation be clipped to a depth landward of the shelf break (Figure 1). This is due to the nature of the Bathymetry Position Index (BPI) calculation, which calculates relative height across the DEM (based on user-defined windows). Where a steep change in gradient occurs, such as the drop-off of a shelf break, this will 'flatten' the remaining features on the shelf, and they may appear smaller in relief relative to the prominent drop-off feature. For optimal performance of the classification toolset, it is recommended the user select a depth threshold landward of the shelf break.

Input parameters

DEM

Input raster dataset (DEM) as a GeoTIFF

DEM Label [Optional]

Label for output directory i.e. PREP_[DEM Label].

If left blank clipped DEM raster dataset is output to origin directory.

Shallow cutoff value (m)

Shallowest depth to clip to as a positive number.

Deepest cutoff value (m) [Optional]

Deepest depth to clip to as a positive number. If left blank maximum depth in existing DEM is used.

Outputs

A clipped GeoTIFF named DEM_[ShallowDepth]m_[DeepDepth]m (i.e. DEM_20m_59m.tif for a DEM clipped below 20m with a maximum depth of 59m) is output.

PREP make DEM from xyz

Converts xyz data to GeoTIFF raster.

If the user has the elevation data in an XYZ format, this tool can grid the XYZ data into an GeoTiFF raster format. A co-ordinate system is required to be specified, which should match the known co-originate system of the input XYZ. This tool requires the XYZ data to be ordered by the Y variable.

Input parameters

Depth (xyz)

Input file with depths as xyz. Data ordered by Y variable.

DEM Label

Label for output DEM and location i.e. test.

Projection (CRS) [Optional]

Projection (CRS) for output DEM. If left blank the CRS of the current project is used.

Convert Depths

Check box to convert postive depths in xyz file to negative (default is no). i.e. xyz output from HYPACK. Leave unchecked if input xyz depth values are negative.

Outputs

A GeoTIFF named DEM_[DEM Label] is written to a directory called PREP_[DEM Label] in the project directory.


Algorithm version: V1.2

Last Modified: 13-Jan-2026


PREP smooth DEM

Iteratively smooth DEM raster dataset.

NOTE: This tool uses the WhiteboxTools - MedianFilter

Smoothing can be performed to reduce noise artefacts in the dataset. This is particularly effective with noise within marine lidar or bathymetry datasets derived from satellite imagery which are more likely to have speckled noise.

The smoothing function performs a 'Median' filter using the Whitebox Tools plugin within QGIS. Median filters were determined to be most effective as they do not include the extremities of data values as would occur with a 'Mean' calculation. Users can input a smoothing iteration. The number of smoothing iterations is added as a suffix to the output dataset e.g. dem_s3 if the DEM was smoothed with three iterations.

Input parameters

DEM

Input raster dataset (DEM) as a GeoTIFF

DEM Label

Label for output directory i.e. PREP_[DEM Label].

If left blank smoothed DEM raster dataset is output to origin directory.

Number of Iterations

Number of iterations to smooth over (i.e. 3).

Outputs

A smoothed GeoTIFF named DEM_s[Number of iterations] (i.e. DEM_s3.tif for 3 iterations) is output.
